# 个人实训总结报告

## 一、项目概述

本次实训中，我独立完成了"岁岁安"微信小程序的开发。这是一款相册管理应用，采用微信云开发架构，实现了用户注册登录、相册管理、照片上传、旅行打卡、回收站管理等核心功能。项目共包含18个页面、4个云函数、4个数据库集合，前后端一体化开发。

## 二、遇到的最大困难及解决过程

### 困难一：云函数部署与环境配置

在项目从本地存储迁移到云开发的过程中，我遇到了最棘手的问题——云函数调用失败，控制台反复报 `DATABASE_COLLECTION_NOT_EXIST` 错误。尽管我已经在云开发控制台创建了数据库集合，但云函数仍然无法访问。

经过反复排查，我发现问题的根源在于云函数的 `cloud.init()` 没有指定环境ID。微信小程序支持多个云开发环境，如果云函数初始化时不指定环境，它可能连接到错误的环境。我在所有4个云函数中显式添加了 `env: 'cloud1-d2gq7l5vl2c158dd6'` 参数，并重新部署后问题得以解决。这个过程让我深刻理解了云开发环境隔离的机制。

### 困难二：图片存储路径失效

另一个重大挑战是图片显示问题。最初上传照片时，我使用的是小程序的临时文件路径（`tempFilePath`），这些路径在小程序重启后就会失效，导致回收站和打卡记录中的图片无法显示。

解决方案是将图片上传到微信云存储，调用 `wx.cloud.uploadFile()` 获取永久的 `cloud://` 链接。虽然旧数据无法恢复，但新上传的图片都能正常显示了。这个经历让我认识到数据持久化的重要性。

### 困难三：Canvas地图交互

在实现旅行打卡的地图功能时，我遇到了SVG不支持和Canvas点击事件被Swiper拦截的问题。最终采用Canvas 2D API绘制省份多边形，并将 `bindtap` 改为 `bindtouchend` 来解决事件冲突，同时使用射线法算法判断点击位置所在的省份。

## 三、对AI辅助编程的心得体会

在本次项目中，我大量使用了Trae AI进行辅助编程，收获颇丰。

**效率提升显著**：AI能够快速生成页面结构、样式代码和云函数逻辑，将原本需要数小时的工作缩短到几分钟。例如，中国地图的Canvas绘制代码、34个省份的坐标数据、云函数的CRUD操作等，AI都能在短时间内给出可用的代码框架。

**但AI不是万能的**：AI生成的代码有时存在逻辑错误或不兼容问题。比如SVG在微信小程序中不支持外部引用、Swiper中tap事件被拦截等问题，AI最初给出的方案并不正确。我需要通过调试和测试来发现这些问题，然后向AI提出更精确的需求，经过多轮迭代才能得到正确的解决方案。

**Prompt工程很重要**：我发现向AI描述需求时越具体，得到的结果越好。模糊的描述如"帮我优化一下"往往得到泛泛的修改，而具体的描述如"回收站改为3列网格布局，相册显示封面图，照片显示缩略图"则能得到精准的代码。学会写好Prompt是有效使用AI的关键技能。

**AI是工具，不是替代品**：AI擅长生成代码片段和解决具体问题，但项目的整体架构设计、技术选型、用户体验优化等仍需要开发者自己的判断。我把AI当作一个高效的编程助手，而不是完全依赖它。

## 四、个人在项目中的具体贡献

作为项目的唯一开发者，我承担了全部工作：

1. **需求分析与设计**：确定了用户系统、相册管理、旅行打卡三大核心模块，设计了数据库结构和页面路由。

2. **前端开发**：完成了18个页面的WXML结构、WXSS样式和JavaScript逻辑，实现了响应式布局和暖棕色系UI主题。

3. **后端开发**：创建了4个云函数，实现了用户认证、相册CRUD、回收站管理、打卡记录等完整的后端API。

4. **特色功能实现**：独立实现了Canvas中国地图绘制、射线法省份检测、Swiper滑动切换等创新交互功能。

5. **问题排查与修复**：解决了云函数环境配置、图片存储路径、事件冲突等多个技术难题。

## 五、总结与反思

通过本次实训，我不仅掌握了微信小程序的开发技术，更重要的是学会了如何有效利用AI工具提升开发效率。AI辅助编程不是简单地复制粘贴代码，而是需要开发者具备足够的技术判断力来审查、调试和优化AI生成的代码。

在未来的学习中，我将继续深入小程序开发技术，同时探索更多AI辅助编程的最佳实践，努力成为一名既懂技术又善用工具的全栈开发者。
